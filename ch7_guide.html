7.1 Trees

Definition — Tree

A tree is a connected (undirected) graph with no cycles. Trees cannot contain loops or multiple edges (these are cycles), so trees are necessarily simple graphs.

Definition — Forest

A graph is a forest if every connected component is a tree.

Theorem — Characterisation of Trees

A graph is a tree if and only if there is a unique simple path between any two of its vertices.

(⇒) Let \(T\) be a tree. By connectivity every two vertices are joined by a path, reducible to a simple path. Suppose two distinct simple paths \(P_1, P_2\) connect \(a\) to \(b\). Let \(k\) be the last index where they agree and \(s,t\) the first indices after \(k\) where they meet again. The segment from \(u_k\) to \(u_s\) via \(P_1\) and back via \(P_2\) forms a cycle — contradiction.

(⇐) If every two vertices have a unique simple path, \(T\) is connected. Any cycle would give two distinct simple paths between vertices on it — contradiction. So \(T\) is acyclic, hence a tree.

Rooted Trees & Terminology

Definition — Rooted Tree

A rooted tree is a tree with one designated root vertex; all edges are directed away from the root.

  • Parent / Child: If edge \(\overline{uv}\) exists with \(u\) above \(v\), then \(u\) is the parent of \(v\); \(v\) is a child of \(u\).
  • Siblings: Vertices sharing the same parent.
  • Ancestor / Descendant: \(u\) is an ancestor of \(v\) (and \(v\) a descendant of \(u\)) if there is a directed path from \(u\) down to \(v\) (\(u \ne v\)).
  • Leaf: A vertex with no children. Internal vertex: A vertex that is not a leaf.
  • Subtree rooted at \(u\): \(u\) together with all its descendants.
Example — \(T_2\) with root \(c\)
  • Leaves: \(f, g, d, e\). Internal vertices: \(a, b, c\).
  • \(b\) and \(d\) are children of \(a\); they are siblings; \(a\) is their parent.
  • \(a, c\) are ancestors of \(d\); \(b, d, f, g\) are descendants of \(a\).
Definition — \(m\)-ary Trees
  • \(m\)-ary tree: each internal vertex has at most \(m\) children.
  • Full \(m\)-ary tree: each internal vertex has exactly \(m\) children.
  • Binary tree: \(m = 2\). Ordered rooted tree: children of each vertex are linearly ordered (left to right).

7.1 Properties

Theorem 1 — Degree-1 Vertices

A tree with \(n \ge 2\) vertices has at least two vertices of degree 1.

Take a maximum-length simple path \(v_0v_1\cdots v_m\). Suppose \(\deg(v_0) \gt 1\). Then \(v_0\) is adjacent to some \(x \ne v_1\). If \(x = v_i\) (\(i \ge 2\)), there is a cycle — contradiction. Otherwise \(xv_0\cdots v_m\) is longer — contradiction. So \(\deg(v_0) = 1\). Same argument gives \(\deg(v_m) = 1\). These are distinct since \(m \ge 1\).

Theorem 2 — Removing a Leaf

If \(v\) is a degree-1 vertex in a tree \(T\) with \(n \ge 2\), then \(T - v\) is also a tree.

Acyclic: Any cycle in \(T-v\) is a cycle in \(T\) — contradiction.

Connected: For any \(x,y \ne v\), the unique simple path in \(T\) cannot pass through \(v\) as an interior vertex (since \(\deg(v)=1\)), so it remains valid in \(T-v\).

Theorem 3 — Edge Count

A tree with \(n\) vertices has exactly \(n-1\) edges.

Induction on \(n\). Base (\(n=1\)): 0 edges = \(1-1\). ✓

Inductive step: Assume every \(k\)-vertex tree has \(k-1\) edges. Let \(T\) have \(k+1\) vertices. By Thm 1, \(\exists\) degree-1 vertex \(v\). By Thm 2, \(T-v\) is a tree with \(k\) vertices, so it has \(k-1\) edges (IH). Adding back \(v\) and its one edge: \(T\) has \(k = (k+1)-1\) edges. ✓

Theorem 4 — Full \(m\)-ary Vertex Count

A full \(m\)-ary tree with \(i\) internal vertices has \(n = mi + 1\) total vertices.

Every vertex except the root is a child of exactly one internal vertex. Each internal vertex has exactly \(m\) children, giving \(mi\) children total. Adding the root: \(n = mi + 1\).

Theorem 5 — Full \(m\)-ary Formulas

For a full \(m\)-ary tree with \(n\) total vertices, \(i\) internal vertices, and \(\ell\) leaves:

\[n = mi+1 = \frac{m\ell-1}{m-1},\quad i = \frac{n-1}{m} = \frac{\ell-1}{m-1},\quad \ell = \frac{(m-1)n+1}{m} = (m-1)i+1\]

Example — Chain Letter

Each person sends to 4 others or stops. 100 people received it but did not send it on (\(\ell = 100\), \(m = 4\)):

\[n = \frac{m\ell-1}{m-1} = \frac{400-1}{3} = 133\]
Definitions — Level, Height, Balanced
  • Level of \(v\): length of the path from the root to \(v\). Root has level 0.
  • Height: maximum level over all vertices.
  • Balanced: all leaves are at level \(h\) or \(h-1\) (where \(h\) is the height).
Theorem 6 — Vertex Bound by Level

In an \(m\)-ary tree there are at most \(m^k\) vertices at level \(k\), and at most \(m^h\) leaves (where \(h\) = height).

Theorem 7 — Height Lower Bound

An \(m\)-ary tree with \(\ell\) leaves satisfies \(h \ge \lceil\log_m \ell\rceil\). If the tree is full and balanced, \(h = \lceil\log_m \ell\rceil\).

7.2 Applications

Binary Search Trees (BST)

BST Construction Algorithm
  1. Assign the first item to the root.
  2. To insert a new item: compare with the current node; if smaller, go left; if larger, go right.
  3. Continue until a null child is reached; place the item there.
Example — Lexicographic BST

Insert in order: mathematics, physics, geography, zoology, meteorology, geology, psychology, chemistry.

  • Root: mathematics
  • physics > mathematics → right: physics
  • geography < mathematics → left: geography
  • zoology > mathematics, > physics → right of physics: zoology
  • meteorology > mathematics, < physics → left of physics: meteorology
  • geology < mathematics, > geography → right of geography: geology
  • psychology > mathematics, < physics, > meteorology → right of meteorology: psychology
  • chemistry < mathematics, < geography → left of geography: chemistry
Remark

Maximum comparisons to find an item = height of the BST. Balanced BSTs minimise search time.

Universal Address System

Definition
  1. Label the root 0.
  2. Label its children \(1, 2, \ldots\) left to right.
  3. A vertex at level \(n\) with label \(A\) has children \(A.1, A.2, A.3, \ldots\)

Lexicographic ordering: \(x_1.x_2\cdots.x_n \lt y_1.y_2\cdots.y_m\) if the first differing position satisfies \(x_i \lt y_i\), or if \(x\) is a proper prefix of \(y\).

Example Address Order

\(0 \lt 1 \lt 1.1 \lt 1.2 \lt 1.3 \lt 2 \lt 3 \lt 3.1 \lt 3.1.1 \lt 3.1.2 \lt 3.1.2.1 \lt \cdots \lt 3.2 \lt 4 \lt 5.1 \lt 5.1.1 \lt 5.2 \lt 5.3\)

Ancestors of \(3.1.2.3\) are: \(3.1.2, 3.1, 3, 0\).

7.3 Spanning Trees

Definition — Spanning Tree

A spanning tree of a simple graph \(G\) is a subgraph that is a tree containing every vertex of \(G\).

Theorem — Spanning Tree Existence

A simple graph is connected if and only if it contains a spanning tree.

(⇐) A spanning tree is connected and contains all vertices, so \(G\) is connected.

(⇒) If \(G\) is connected and acyclic, \(G\) itself is a spanning tree. Otherwise, remove any edge from a cycle (connectivity is preserved since the other side of the cycle remains). Repeat until acyclic. The result spans all vertices and is acyclic and connected — a spanning tree.

Depth First Search (DFS)

DFS Algorithm
  • Step 1. Choose any vertex, label it 1, add to \(T\).
  • Step 2. From vertex \(j\): choose an unlabelled neighbour, label it \(j+1\), add vertex and edge to \(T\). If all neighbours labelled, backtrack to vertex \(j-1\).
  • Stop when all vertices are labelled. \(T\) (rooted at vertex 1) is the DFS spanning tree.
Example — DFS on \(G\) (edges: \(\{ab,ae,bf,cd,cg,dg,ef,eg,fg\}\))

Alphabetical DFS from \(a\): \(a(1) \to b(2) \to f(3) \to e(4) \to g(5) \to c(6) \to d(7)\). Tree edges: \(ab, bf, fe, eg, gc, cd\).

Non-alphabetical DFS from \(a\): \(a(1) \to e(2) \to f(3) \to g(4) \to d(5) \to c(6) \to b(7)\). Tree edges: \(ae, ef, fg, gd, dc, fb\).

Breadth First Search (BFS)

BFS Algorithm
  • Step 1. Choose any vertex as root, label it 1, add to \(T\).
  • Step 2. Visit labelled vertices in ascending order. For vertex \(j\) being visited, label all unlabelled neighbours consecutively and add those edges to \(T\).
  • Stop when all vertices are labelled.
Example — BFS on \(G\)

Alphabetical BFS from \(a\): \(a(1),b(2),e(3),f(4),g(5),c(6),d(7)\). Tree edges: \(ab, ae, bf, eg, gc, gd\).

7.4 Minimum Spanning Trees

Definitions
  • Weighted graph: each edge \(e\) has a real-valued weight \(w(e)\). Total weight \(w(G) = \sum w(e)\).
  • Minimum spanning tree (MST): a spanning tree with the least total weight. Both algorithms below are greedy algorithms.

Prim's Algorithm

Prim's Algorithm
  1. Choose any edge of minimum weight; add to \(T\).
  2. Among edges joining a vertex in \(T\) to one not in \(T\), choose minimum weight; add to \(T\).
  3. Stop when \(T\) has \(n-1\) edges.

Kruskal's Algorithm

Kruskal's Algorithm
  1. Sort all edges by increasing weight. Let \(T\) = isolated vertices.
  2. Examine edges in order. Add an edge if it joins two distinct connected components (no cycle created). Otherwise skip it.
  3. Stop when \(T\) has \(n-1\) edges.
Example — 12-vertex Grid Graph

Prim's order: \(bf(1), ab(2), jf(2), ae(3), bc(3), cd(1), cg(2), jk(3), kl(1), lh(3), ij(3)\). Total weight = 24.

Kruskal's order: \(bf(1), cd(1), kl(1), ab(2), cg(2), fj(2), bc(3), ae(3), hl(3), ij(3), jk(3)\). Edge \(fg\) skipped (would create cycle). Total weight = 24.

Quiz

Q1. A tree with \(n\) vertices has exactly how many edges?

C. By the edge-count theorem (induction using Thms 1 & 2): \(|E| = |V| - 1\).

Q2. Which is equivalent to “\(G\) is a tree”?

C. The theorem states: tree ⇔ unique simple path between every pair of vertices. A should say \(n-1\) edges; B is self-contradictory; D is a forest, not a tree.

Q3. A full 3-ary tree has \(i=4\) internal vertices. Find \(n\).

B. \(n = mi+1 = 3(4)+1 = 13\). Leaves: \(\ell = (m-1)i+1 = 2(4)+1 = 9\).

Q4. A full binary tree has \(i=6\) internal vertices. How many leaves?

C. For \(m=2\): \(\ell = (m-1)i+1 = 1\cdot6+1 = 7\).

Q5. Vertices sharing the same parent are called:

C. Siblings share the same parent. Ancestors are vertices above on the path to root; descendants are below.

Q6. A full balanced binary tree with \(\ell=8\) leaves. What is its height?

B. \(h = \lceil\log_2 8\rceil = \lceil3\rceil = 3\).

Q7. In DFS, when all neighbours of vertex \(j\) are already labelled, you:

C. Backtracking to \(j-1\) is the defining feature of DFS that creates the depth-first exploration.

Q8. In Prim's algorithm, at each step you add:

C. Prim's always extends the tree by the cheapest edge crossing the cut between \(T\) and \(V \setminus T\).

Q9. In Kruskal's, an edge is skipped when:

A. Adding such an edge would create a cycle. Kruskal's only adds edges that merge two distinct components.

Q10. A simple graph \(G\) is connected if and only if:

C. Spanning tree existence theorem. A is not sufficient (a disconnected graph can have all degrees ≥ 2); B is not sufficient for general graphs.
